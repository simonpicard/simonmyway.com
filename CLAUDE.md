## Commands

- **Dev server:** `npm run dev`
- **Build:** `npm run build`
- **Lint (JS/TS):** `npm run lint`
- **Lint (Python):** `ruff check scripts/` and `ruff format --check scripts/`
- **Python scripts:** `uv sync` then `uv run scripts/<script>.py` (Python 3.11). CI still uses `pip install -r requirements.txt` — keep both dependency lists in sync.

There are no test suites in this project.

## Architecture

This is a personal website (simonmyway.com) built with Next.js (App Router), TypeScript, and Tailwind CSS v4. Dark theme only.

### Internationalization

The site is bilingual (English + French). All pages live under `src/app/[locale]/` (`en` and `fr`); the root layout is `src/app/[locale]/layout.tsx`. Legacy unprefixed URLs (`/blog/:slug`, `/about`, ...) permanently redirect to `/en/...` via `next.config.js` — these redirects must never be removed. UI strings live in `src/lib/i18n.ts` (`dictionaries`); hreflang alternates are emitted per page and in the sitemap.

### Content pipeline

- Blog posts are markdown files in `content/posts/en/` and `content/posts/fr/`, named `YYYY-MM-DD-title.md`. A translation shares its filename with the original — that filename is the key linking the two versions (hreflang, language switcher). A post missing from `fr/` simply doesn't appear on the French site.
- Frontmatter is parsed with gray-matter; titles are extracted from the first `# heading` if not in frontmatter. Slugs are derived from the localized title via `slugify()` (accents are transliterated), so French posts get French slugs. Never change `slugify()` in a way that alters existing slugs — URLs would break.
- Videos are fetched from the YouTube API and cached in `content/videos.json` with a 1-day TTL. Requires `YOUTUBE_CHANNEL_ID` and `YOUTUBE_API_KEY` env vars.
- RSS feeds are served at `/en/feed.xml` and `/fr/feed.xml` (Atom format) with full HTML content generated by markdown-it + highlight.js with inline CSS styles for email compatibility.

### Newsletter automation

A GitHub Actions workflow (`newsletter.yml`) runs daily at 10 AM CET:
1. `scripts/check_feed.py` — fetches both live RSS feeds (en + fr), finds entries published yesterday, outputs them as JSON tagged with their language
2. `scripts/send_newsletter.py` — creates and sends Mailchimp campaigns for each new entry, targeted by language: subscribers with the `LANGUAGE` merge field set to `fr` get French campaigns, everyone else gets English ones (merge tag configurable via `MAILCHIMP_LANGUAGE_MERGE_TAG`)

### Key lib modules

- `src/lib/posts.ts` — reads/parses markdown posts from disk, provides `getAllPosts(locale)`, `getPostBySlug(locale, slug)`, `getTranslation()` and `getBlogPathMap()`
- `src/lib/i18n.ts` — locale list and UI-string dictionaries
- `src/lib/markdown.ts` — server-side markdown-to-HTML conversion (used for RSS feed content); converts highlight.js classes to inline styles
- `src/lib/videos.ts` — YouTube API fetching with file-based caching

### Rendering

Blog posts render client-side using `react-markdown` + `remark-gfm` + PrismJS for syntax highlighting (see `blog/[slug]/page.tsx`). The RSS feed uses a separate server-side rendering path via markdown-it + highlight.js (`src/lib/markdown.ts`).

### Path alias

`@/*` maps to `./src/*` (configured in tsconfig.json).
